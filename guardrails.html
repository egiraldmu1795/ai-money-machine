Guardrails & safety · how it can't go rogue

It runs itself — but it can't move money or publish without you

The honest objection to any "AI money machine" is: what stops it from quietly doing something irreversible? The answer is a single safety boundary, enforced in code. Everything internal runs full-speed and autonomous. The small set of actions that move real value or go public default to off — and a human pulls the trigger.

The one rule

Safe-by-default, enforced at one chokepoint

Not "remember to be careful." A single piece of code every action passes through.

  • Everything internal — build, backtest, paper-trade, draft content, write local files — runs full-speed and autonomous.
  • The guardrail exists only around actions that move real value or go public and cannot be undone.
  • For those, the system defaults to dry-run / preview, and a human must pull the live trigger.
  • One classification + one audit row per execution. Fail closed — anything it can't classify is blocked, not allowed.

Gate 1

The Money Gate

Before any real-money action: a live order, a transfer, a payout.

  • Dry-run is the default. The system proposes and simulates; nothing hits a broker on its own.
  • Going live needs an explicit, per-run, human-initiated approval token — never a global flag left on.
  • Amount caps: a notional ceiling above which even an approved action is refused.
  • The trading system is paper-first; you decide if real money is ever involved, at your own risk.

Gate 2

The Publish Gate

The same discipline for anything that goes outward.

  • Preview is the default. A public git push, a social/video post, an external message — all draft-first.
  • A human approves before it ships; destinations are allowlisted (no posting to an unknown place).
  • AI drafts, you approve, then it goes out — never auto-publish under your name.

Gate 3

The Sensitive-Action Policy

The contract the gatekeeper enforces.

  • Classifies sensitive actions into clear classes (money, publish, secrets/PII, destructive).
  • Defines how each class is gated and which safe mode it defaults to.
  • Specifies what gets logged — an append-only audit row so every gated action is accountable.
  • One chokepoint, no side doors: every executable action passes through the same gate.

Common questions

Can the AI move real money on its own?

No. Every real-money action passes through the money gate, which defaults to dry-run. Going live requires an explicit per-run human approval — there is no flag you leave on.

Can it publish or post under my name without me?

No. Outward actions pass through the publish gate (preview by default); a human approves before anything ships, to allowlisted destinations only.

Is the safety just "be careful," or is it real?

It's enforced in code on the real execution path — classify → safe-by-default → fail closed → human approval → append-only audit. No side doors. (ADR-0004 + ADR-0003.)
