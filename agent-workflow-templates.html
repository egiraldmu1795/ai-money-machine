Research fan-out + adversarial verify

A single confident generation is how you get a plausible answer that's quietly wrong. This template is the fix: research every angle in parallel, then send a skeptic after each claim before it's allowed into the answer. Copy it. Run it.

The pattern

Decompose a question into sub-queries, research each in parallel, then send a skeptic after every claim before it's allowed into the answer. The verify step is what separates this from a glorified search: one agent's plausible-but-wrong finding gets refuted by an independent agent whose only job is to refute it.

Use it when

You need an answer you can stand behind — a market read, a technical decision, a "is this actually true" check — and a single confident generation isn't enough.

The runnable script

This runs in an orchestration harness exposing agent(), pipeline(), and parallel() (the same shape the AI Money Machine course builds). Swap those for your framework's primitives and the logic carries over unchanged — the recipe below is framework-agnostic.

// Research a question with cited, adversarially-verified claims.
const QUESTION = "What pricing model do successful solo-dev SaaS tools use in 2026?"

const CLAIM_SCHEMA = { type: "object", properties: {
  claims: { type: "array", items: { type: "object", properties: {
    text: { type: "string" }, source: { type: "string" }
  }, required: ["text", "source"] } }
}, required: ["claims"] }

const VERDICT_SCHEMA = { type: "object", properties: {
  refuted: { type: "boolean" }, reason: { type: "string" }
}, required: ["refuted", "reason"] }

// 1. Fan out into angles (decompose once, then research each in parallel).
const angles = [
  "pricing tiers and price points",
  "free trial vs freemium conversion",
  "one-time vs subscription vs usage-based",
]

// 2. Pipeline: research each angle, then verify each claim it produced — no barrier,
//    so angle A's claims verify while angle C is still being researched.
const results = await pipeline(
  angles,
  angle => agent(
    `Research "${QUESTION}" focused on: ${angle}. Return only claims you can attribute to a named source.`,
    { schema: CLAIM_SCHEMA, label: `research:${angle}` }
  ),
  research => parallel(research.claims.map(c => () =>
    agent(
      `Try to REFUTE this claim. Default to refuted=true if you cannot find independent support: "${c.text}" (source: ${c.source})`,
      { schema: VERDICT_SCHEMA, label: "verify" }
    ).then(v => ({ ...c, refuted: v.refuted, reason: v.reason }))
  ))
)

const confirmed = results.flat().filter(Boolean).filter(c => !c.refuted)
log(`${confirmed.length} claims survived verification`)
return { question: QUESTION, confirmed }

Framework-agnostic recipe

  1. Split the question into 3–5 non-overlapping research angles.
  2. For each angle, ask an agent for claims with named sources only (no unsourced assertions).
  3. For each claim, ask a fresh agent to refute it — prompt it to default to "refuted" when it can't find independent support.
  4. Keep only claims that survive. Synthesize the answer from those.

Tuning notes

  • Skeptics must default to refuted. A verifier prompted neutrally rubber-stamps; prompted to refute-unless-proven, it catches the hallucinations. This single framing is most of the value.
  • One skeptic is usually enough. For high-stakes claims run 3 and require a majority to refute — cheap insurance against a bad verifier.
  • Pipeline, don't barrier. There's no reason a fast angle's claims should wait for a slow angle to finish researching before they verify.
  • Scale angles to question breadth; scale skeptics-per-claim to stakes.

This is exactly how the EAB trading committee vets a thesis before it reaches the human money-gate — a researcher proposes, a risk and a bear agent try to tear it down, and only what survives gets sized. The pattern is general: any time a wrong answer is expensive, verify before you commit.
